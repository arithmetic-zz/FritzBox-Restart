# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A small CLI tool that reboots an AVM FRITZ!Box router via one of two methods. No build, no test suite — three standalone Python scripts plus a thin dispatcher.

## Commands

```bash
pip install -r requirements.txt
playwright install chromium          # only for the web method

python3 fritzbox_reboot.py           # interactive method picker
python3 fritzbox_reboot.py --method api   # TR-064 directly
python3 fritzbox_reboot.py --method web   # Playwright directly
python3 fritzbox_reboot_api.py       # standalone, bypasses dispatcher
python3 fritzbox_reboot_web.py       # standalone, bypasses dispatcher
```

Requires a `.env` (see `.env.example`); `FRITZBOX_PASSWORD` is the only required var. Every entrypoint prompts for a typed `yes`/`y` confirmation before doing anything.

## Architecture

`fritzbox_reboot.py` is a dispatcher only — it does the version header, confirmation prompt, and method selection, then imports `reboot_fritzbox()` from one of the two backend modules and wraps the call in module-specific exception handling. The backends do **not** import the dispatcher.

Each backend (`fritzbox_reboot_api.py`, `fritzbox_reboot_web.py`) is self-contained and runnable on its own: it loads `.env` at import time, defines its own `print_header()`/`step()` helpers, exposes `reboot_fritzbox()` for the dispatcher, and has a full `__main__` block duplicating the confirm-and-run flow. This duplication is intentional so each script works standalone.

- **API backend** — calls `DeviceConfig1:Reboot` via `fritzconnection` (TR-064 SOAP, HTTP Digest auth). Also has `generate_curl_script()`, which writes a self-contained `reboot_fritzbox.sh` with credentials inlined and the same SOAP body hand-rolled for `curl --digest`. Requires TR-064 ("Zugriff für Anwendungen") enabled on the box.
- **Standalone Windows script** — `reboot_fritzbox.ps1` is a hand-maintained PowerShell port of the curl script (same TR-064 SOAP `Reboot`, Digest auth via `System.Net.CredentialCache`). It is **not** generated by the Python tool; keep it in sync manually if the SOAP call changes. It ships with placeholders (committed, no real creds); `reboot_fritzbox.cmd` is a double-click launcher that runs it with `-ExecutionPolicy Bypass`.
- **Web backend** — Playwright drives Chromium (`headless=False`) through login → System → Sicherung → "FRITZ!Box neu starten" card → "Neu starten" confirm. **Safety invariant: it matches the exact German UI text of the reboot card and confirm button and aborts on any mismatch — it must never click factory-reset, backup, or restore controls.** Preserve this when editing selectors.

## Conventions

- Version and tested-OS strings are duplicated as `VERSION` / `TESTED_FRITZOS` constants in all three files plus the README header — update all of them together.
- UI strings are English; FRITZ!Box web selectors necessarily use the German labels from the FritzOS interface.
- `.env` and the generated `reboot_fritzbox.sh` hold plaintext passwords and are gitignored — never commit either.
- Tested only against FRITZ!Box 7590 on FritzOS 8.21 / 8.25.
